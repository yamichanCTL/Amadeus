## 文件识别交互

侧栏入口统一命名为“语音识别”。选择或拖放文件后，页面只保存本地待处理文件并显示文件名；只有点击“确认并开始识别”后才创建 `/v1/transcribe` 请求。“取消”会清空待处理文件，不上传数据。

页面不再展示“最近任务”卡片；已完成结果仍写入历史记录页。录音识别和实时识别入口保持在底部操作栏。

页面采用单列宽卡片结构：文件上传区位于顶部，“识别预览”紧随其下并横向展示识别文本、复制/润色/翻译操作和音频播放器；识别设置与助手区域排列在预览之后。该结构避免宽屏下右侧预览过窄、左侧产生大面积空白。

## 实时识别连接

实时识别优先连接设置中的后端 `/v1/stream`。当前页面运行在 HTTP/HTTPS 来源时，还会把同源 `/v1/stream` 加入候选列表，从而在公网直连失败时使用 Vite WebSocket proxy。第一次连接的 `error/close` 不会提前中断第二个候选；只有全部候选地址都失败后才显示错误，并列出实际尝试地址。

后端首次创建流式 session 时可能需要加载 FireRed VAD。该同步初始化现在通过 `asyncio.to_thread()` 运行，WebSocket `accept` 后先让事件循环刷新握手，因此不会再被前端 5 秒连接计时器关闭。开发运行建议关闭 Uvicorn `--reload`，避免大型模型/重载工作子进程退出后只剩监听父进程。

页面会依次显示“正在连接”“连接成功，等待模型加载”“模型已加载，正在预热”“正在监听”。客户端只在 `configured` 后打开 PCM 采集。点击停止会立即停止采集、关闭 WebSocket、归档已收到文本并恢复 `idle`，关闭事件采用幂等处理，不再等待远端延迟关闭回调。

## 录音上传与错误处理

普通录音优先通过 Web Audio 采集连续 PCM，并在停止时封装为 `audio/wav` 后与文件识别共用 `POST /v1/transcribe`；`MediaRecorder` 的 WebM/Opus 只作为兜底。应用级 `speechRecorder` 会预热当前真实麦克风，触发后立即使用已存活轨道，录音状态在触发开始时就进入 `recording`，因此再次按触发键会走停止路径。离线录音始终独立采集所选实体麦克风，即使音频中转正在运行也不读取中转轨道；只有显式选择“扬声器（系统音频输出）”时才走 loopback。回环、monitor、stereo mix 和 `CABLE Output` 等虚拟输出输入会被拒绝。麦克风启动和停止都有超时兜底，失败会清理麦克风轨道、隐藏状态浮窗并恢复空闲状态；网络层 `Failed to fetch` 会显示实际 ASR 请求目标，并提示检查后端、后端地址、HTTPS/HTTP 混合内容和 CORS。

录音浮窗位于屏幕中下方，普通状态尺寸为 200×32。采集阶段只显示“语音输入中”，并把真实 RMS/peak 连续追加到 28 段时间历史波形；提交后循环显示 `thinking.` / `thinking..` / `thinking...`。处理中再次按全局触发键或点击“强制停止”，会同步停止录音器、HTTP 请求/轮询、已知后端任务和 WebSocket，并立即恢复可再次开始状态。

自动输入在录音浮窗显示前先捕获当前前台窗口，ASR 最终文本返回后立即启动投递，不等待前端历史记录、归档或延时统计。Electron 主进程会先恢复捕获到的窗口，再检查焦点是否为可编辑文本控件；确认可编辑后才写剪贴板并发送标准 Ctrl+V，成功后直接关闭 Thinking 浮窗，不生成结果框。无法输入时才复用 Thinking 浮窗展示 ASR 结果，并提供“复制”和“×”按钮。详见[输入、浮窗与跨应用注入](INPUT_AND_OVERLAYS.md)。

模型管理中的参考文本异步任务会先立即查询一次，随后以 100/250/500 ms 自适应轮询，不再固定先等待 1 秒。变声器的语音模式将 ASR 与 TTS 拆成两个请求，ASR 文本先回填，再等待 TTS；自动化门槛为从 220 ms ASR 模拟响应到 DOM 可见总耗时小于 500 ms。

设置中的任务超时默认 20 秒，并随每次 `/v1/transcribe` 请求发送到后端；后端同步和异步 ASR 推理都使用同一限制。`0` 表示不限制，首次加载大型模型时可按实际机器性能调高。

## 实时预览与字幕控制

识别预览压缩为秒级时间范围和当前文本，不再重复日期、类型和“识别结果”标签。partial 即使早于 `speech_start` 到达也会创建当前条目，并始终覆盖最新结果。

字幕浮窗右上角提供关闭和设置按钮。关闭只隐藏字幕、不停止实时识别；设置按钮唤醒主窗口并进入设置页。设置页支持显示开关、字号、颜色、透明度、宽高、位置恢复和即时预览。

已有 SQLite 数据库仍包含早期的 `asr_tasks.diarize_enabled NOT NULL` 物理列。说话人分离 API 已删除，但 ORM 暂时保留该列作为兼容字段，创建任务时固定写入 `false`，避免旧数据库在录音上传时触发完整性错误。后续只有在引入正式 schema migration 后才能删除该物理列。

## 标点恢复

“标点恢复”开关通过 `enable_punctuation` 传给后端。开启后使用 FunASR `ct-punc` CT-Transformer：

- 模型首次使用时由 FunASR 下载并缓存，之后进程内复用同一实例；
- 推理放在线程中，避免阻塞 FastAPI 事件循环；
- 模型缺失或输出无效时返回明确错误，不再静默把原文当作已处理结果；
- 单段识别结果会同步更新 segment 文本，避免全文和时间段文本不一致。

可通过 `PUNCTUATION_MODEL` 和 `PUNCTUATION_DEVICE` 覆盖模型标识与运行设备。

## 已移除能力

说话人分离当前不在产品范围内。桌面开关、请求字段、响应 speaker 字段、后端占位处理器和任务调用链均已删除；数据库兼容列不代表该能力重新开放。

## 任务级延时

确认文件后创建独立 trace，记录上传发送、响应/入队、模型就绪、ASR、标点、热词、LLM 后处理、持久化、前端归档与完成。开发调试台以瀑布图展示各阶段，并保留原始事件表和 JSON 导出。
