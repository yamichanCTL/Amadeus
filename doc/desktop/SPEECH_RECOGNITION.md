## 文件识别交互

侧栏入口统一命名为“语音识别”。选择或拖放文件后，页面只保存本地待处理文件并显示文件名；只有点击“确认并开始识别”后才创建 `/v1/transcribe` 请求。“取消”会清空待处理文件，不上传数据。

页面不再展示“最近任务”卡片；已完成结果仍写入历史记录页。录音识别和实时识别入口保持在底部操作栏。

页面采用单列宽卡片结构：录音控制和识别预览位于顶部，识别文本、复制、“使用当前 Prompt”和音频播放器共用结果卡片；识别设置随后排列，文件上传区位于底部。无功能的人物助手展示区已删除，空间只保留给可操作内容。

## 实时识别连接

实时识别优先连接设置中的后端 `/v1/stream`。当前页面运行在 HTTP/HTTPS 来源时，还会把同源 `/v1/stream` 加入候选列表，从而在公网直连失败时使用 Vite WebSocket proxy。第一次连接的 `error/close` 不会提前中断第二个候选；只有全部候选地址都失败后才显示错误，并列出实际尝试地址。

后端首次创建流式 session 时可能需要加载 FireRed VAD。该同步初始化现在通过 `asyncio.to_thread()` 运行，WebSocket `accept` 后先让事件循环刷新握手，因此不会再被前端 5 秒连接计时器关闭。开发运行建议关闭 Uvicorn `--reload`，避免大型模型/重载工作子进程退出后只剩监听父进程。

页面会依次显示“正在连接”“连接成功，等待模型加载”“模型已加载，正在预热”“正在监听”。客户端只在 `configured` 后打开 PCM 采集。每个 PCM block 在发送 WebSocket 的同时复制到会话录音缓冲；点击停止会立即停止采集、关闭 WebSocket、把完整 PCM 封装为本机 WAV、归档 JSON/文本并恢复 `idle`，关闭事件采用幂等处理，不会重复写文件。

## 录音上传与错误处理

普通录音优先通过 Web Audio 采集连续 PCM，并在停止时封装为 `audio/wav` 后与文件识别共用 `POST /v1/transcribe`；`MediaRecorder` 的 WebM/Opus 只作为兜底。应用级 `speechRecorder` 会预热当前真实麦克风，触发后立即使用已存活轨道。离线/TTS 单次录音关闭浏览器 AEC、降噪和自动增益；Worklet 用音频帧位置检测并补齐缺帧时间轴，避免 WAV 被压短后出现卡顿跳变。录音状态在触发开始时就进入 `recording`，因此再次按触发键会走停止路径。离线录音始终独立采集所选实体麦克风，即使音频中转正在运行也不读取中转轨道；只有显式选择“扬声器（系统音频输出）”时才走 loopback。回环、monitor、stereo mix 和 `CABLE Output` 等虚拟输出输入会被拒绝。麦克风启动和停止都有超时兜底，失败会清理麦克风轨道、隐藏状态浮窗并恢复空闲状态；网络层 `Failed to fetch` 会显示实际 ASR 请求目标，并提示检查后端、后端地址、HTTPS/HTTP 混合内容和 CORS。

录音浮窗位于屏幕中下方，普通状态尺寸为 260×42。采集阶段中间显示真实 RMS/peak 时间历史波形，左右分别为 `×` 取消和 `✓` 提交；取消会中止录音/请求，提交会停止录音并进入识别与自动输入。提交后循环显示 `thinking.` / `thinking..` / `thinking...`，处理中仍可通过左侧 `×`、全局触发键或主界面“强制停止”立即恢复。

ASR 最终文本返回后先同步写入 Amadeus 的当前结果和历史状态，再并行执行本机归档与跨应用投递。因此即使 Windows UIAutomation 卡住，软件内结果区也已经可见和可复制。实时识别每次收到 `final` 事件也会立即刷新当前结果，不必等到手动停止。开启自动润色时，投递优先选择 `llm_outputs` 中的增强文本，增强失败才回退原始 ASR。详见[输入、浮窗与跨应用注入](INPUT_AND_OVERLAYS.md)。

“实时对话”的单次语音按钮停止录音并完成 ASR 后，会把识别结果自动填入本软件的消息输入框；已有草稿时在下一行追加，用户可继续修改后再点击发送，不再跳过输入框直接触发 Agent。免按键监听仍保持自动提交，满足连续对话场景。

AI 润色成功时，后端同步/异步任务日志会记录任务 ID、润色后字符数和最终润色文本，便于确认 ASR 后处理结果；Token 和 LLM 请求凭据不会进入该日志。服务端音频/JSON 调试归档为 opt-in，普通和实时识别都只有在“允许服务端保存调试数据”开启时才留存。

## LLM Prompt 卡片

前端不再维护两套重复的模型连接配置。模型管理的“LLM 设置”只负责厂商、接口、模型、Token、目标语言和风格；Prompt 编辑统一放在语音识别页。

语音识别页内置“ASR 润色”“翻译为英文”“会议纪要”卡片。点击卡片会立即把它设为当前 Prompt，自动增强和手动“使用当前 Prompt”共用该值。卡片支持新增、重命名、编辑、保存和删除；至少保留一张卡片。store v36 会把旧 `llmPolishPrompt` 迁移为默认“ASR 润色”卡片，不丢失用户已有 Prompt。

选择翻译或其他功能卡片时仍通过自定义 Prompt 执行，结果保存在兼容的 LLM 输出字段中；历史数据中的旧润色/翻译响应仍可读取。

两次离线识别紧邻时，第二次注入不再等待卡死的第一轮 helper：最新请求会取消旧注入并跳过未开始的中间项，旧 helper 的退出回调也不能清理新请求。即时后端响应的自动化门槛是每轮“上传请求发送→自动回填完成”低于 500 ms；专项脚本覆盖卡死首轮、连续双次和 30 轮压力测试。

模型管理中的参考文本异步任务会先立即查询一次，随后以 100/250/500 ms 自适应轮询，不再固定先等待 1 秒。变声器的语音模式将 ASR 与 TTS 拆成两个请求，ASR 文本先回填，再等待 TTS；自动化门槛为从 220 ms ASR 模拟响应到 DOM 可见总耗时小于 500 ms。

设置中的任务超时默认 20 秒，并随每次 `/v1/transcribe` 请求发送到后端；短音频同步推理继续直接使用该限制。超过后端同步时长阈值的长音频会进入 Celery 队列，后端按“音频时长的 3 倍加 60 秒”扩大执行预算（最高 3600 秒，`0` 仍表示不限制），renderer 对异步任务至少轮询 30 分钟，不再在 20 秒后提前报告超时。真实长音频仍受 500 MiB 上传上限、Celery worker 和所选模型能力约束。

## 实时预览与字幕控制

识别预览压缩为秒级时间范围和当前文本，不再重复日期、类型和“识别结果”标签。partial 即使早于 `speech_start` 到达也会创建当前条目，并始终覆盖最新结果。

字幕浮窗右上角提供关闭和设置按钮。关闭只隐藏字幕、不停止实时识别；设置按钮唤醒主窗口并进入设置页。设置页按“常规 / 音频 / 识别与字幕 / 数据与隐私”分页；点击“预览字幕框”后，字号、颜色、透明度、宽度、高度和位置的每次变化都会立即重新配置已打开的预览框，无需反复点击预览。

已有 SQLite 数据库仍包含早期的 `asr_tasks.diarize_enabled NOT NULL` 物理列。说话人分离 API 已删除，但 ORM 暂时保留该列作为兼容字段，创建任务时固定写入 `false`，避免旧数据库在录音上传时触发完整性错误。后续只有在引入正式 schema migration 后才能删除该物理列。

## 标点恢复

“标点恢复”开关通过 `enable_punctuation` 传给后端。开启后使用 FunASR `ct-punc` CT-Transformer：

- 模型首次使用时由 FunASR 下载并缓存，之后进程内复用同一实例；
- 推理放在线程中，避免阻塞 FastAPI 事件循环；
- 模型缺失或输出无效时返回明确错误，不再静默把原文当作已处理结果；
- 单段识别结果会同步更新 segment 文本，避免全文和时间段文本不一致。

可通过 `PUNCTUATION_MODEL` 和 `PUNCTUATION_DEVICE` 覆盖模型标识与运行设备。

## 已移除能力

说话人分离当前不在产品范围内。桌面开关、请求字段、响应 speaker 字段、后端占位处理器和任务调用链均已删除；数据库兼容列不代表该能力重新开放。

## 任务级延时

确认文件后创建独立 trace，记录上传发送、响应/入队、模型就绪、ASR、标点、热词、LLM 后处理、自动回填开始/完成、持久化、前端归档与完成。开发调试台以瀑布图展示各阶段，并保留原始事件表和 JSON 导出。
