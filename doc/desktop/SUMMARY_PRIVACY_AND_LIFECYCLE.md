## 当日总结

“当日总结”的总结类型是固定选择，不再要求用户填写后端归档目录名：

- `Both / 所有类型` 不传 category，同时收集当日离线与实时 ASR 类别；
- `离线识别` 对应服务端类别 `一段语音转写`；
- `实时识别` 对应服务端类别 `实时转录`。

主动总结的日期默认今天，开始时间默认 `00:00`，结束时间默认打开页面时的本地当前时间。被动总结使用相同默认范围，并持久化用户后续修改。

页面提供“总结 Prompt 卡片”，默认包含当日工作总结、待办与阻塞、决策与结论。卡片支持点击切换、新增、命名、编辑、保存和删除；当前卡片由桌面 store 持久化，主动总结和被动总结共用。后端不会把完整归档 JSON 发送给 LLM：每条记录只输出 `[开始时间-结束时间] label`，label 优先读取 `labels.ai_polished` / `llm_outputs.polish.text`，没有 AI 润色的实时记录才回退 ASR label。用户 ID、模型名、耗时、音频路径和 metadata 均不进入总结输入。

总结表单、加载状态、错误、保存提示和最后一次结果都保存在桌面 store；从“当日总结”切换到“模型管理”等页面后再返回，不需要重新生成。总结正文使用安全的 Markdown 组件渲染标题、列表、强调、代码、引用和链接，模型输出中的原始 HTML 只作为文本显示。

## 仅本机记录的总结流程

主动和被动总结都提供独立的“文本来源”选择：

- `本机记录`：renderer 按日期、类型和时间范围筛选本机历史，并把最小化记录随请求发送；
- `服务端归档`：请求不带 `records`，后端只查询已经留存的服务端归档。服务端从未保存过音频/JSON 时应选择本机记录。

该选择不再与“允许服务端保存调试数据”开关隐式绑定。使用本机记录时只把以下字段临时发送给 `/v1/llm/archive-summary`：

- `started_at` / `ended_at`；
- `category`；
- 优先使用 AI 处理结果的 `text`。

音频、文件路径、任务 ID、设备信息和完整 metadata 不进入请求。后端收到显式 `records` 后只总结这些记录，不读取服务端归档作为回退。

每次主动或被动总结完成后都自动写入 Electron 本机数据目录的 `summary-logs/<YYYY-MM-DD>/summary_<范围>_<生成时间>.md`，同一范围反复生成不会覆盖旧日志。用户仍可使用“另存为”选择单独的 Markdown 文件位置。

## 本机音频与 JSON 目录

设置页只能通过目录选择器修改“本机数据保存目录”；该路径不会写入 `archive_dir` 或任何上传请求。新生成的记录使用以下层级，旧浅层文件不会自动移动或删除：

```text
<本机目录>/
├── wav/
│   ├── 实时识别/<YYYY-MM-DD>/*.wav
│   └── 离线语音识别/<YYYY-MM-DD>/*.<原音频扩展名>
├── json/
│   ├── 实时识别/<YYYY-MM-DD>/*.json
│   └── 离线语音识别/<YYYY-MM-DD>/*.json
└── summary-logs/<YYYY-MM-DD>/*.md
```

实时识别的 16 kHz mono PCM 在发送 WebSocket 的同时保存在会话级缓冲，停止或连接关闭时封装为 WAV；即使没有识别出文本，只要捕获到音频也会生成本机 WAV 和相邻 JSON 元数据。

## 服务端调试数据留存

“允许服务端保存调试数据”是严格 opt-in：

- 普通 `/v1/transcribe` 的同步成功、同步失败、Celery 成功和 Celery 失败路径只有在值为 `true` 时才调用服务端音频/JSON 归档；
- `/v1/stream` 与 `/v1/tts/higgs/stream` 默认 `archive=false`，桌面实时字幕、Agent 免按键和实时 ASR+TTS 都显式发送当前开关；
- 请求省略字段时也按 `false` 处理，不能因旧默认值反向开启留存；
- Electron 用户本机的归档目录不属于“服务端保存”，继续用于本机历史、实时 WAV/JSON、离线音频/JSON、总结日志和导出。

后端任务数据库仍保存完成请求和返回结果所需的运行记录；开关控制的是额外调试音频/JSON 文件归档。

## ASR AI 润色日志

同步和异步离线 ASR 在 AI 润色成功后输出结构化 INFO 日志，包含任务 ID、润色结果字符数和最终润色文本。日志不记录 API Token、Base URL 请求头或完整 LLM 配置。

允许服务端归档时，相邻 JSON 同时保存脱敏后的 `llm_outputs` 和便于总结选择的 `labels`：`labels.asr` 是原始 ASR，`labels.ai_polished` 是 AI 润色结果。Token 不会进入任一字段。

## 关闭与后台运行

设置页提供“关闭窗口后保留后台运行”，默认关闭：

- 未启用时，标题栏“关闭”会调用 `app.quit()`，同时终止托盘、快捷键和文本注入 helper；
- 只有显式启用时才隐藏主窗口并保留托盘后台运行；
- renderer 每次加载和设置变化都会将持久化值同步给 Electron main process，不再读取另一份可能过期的 `preferences.json`。
