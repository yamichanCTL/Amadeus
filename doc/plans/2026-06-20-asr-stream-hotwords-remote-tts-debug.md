# ASR 双通路、离线热词、远程 TTS 与调试台实施计划

日期：2026-06-20

## 目标

- ASR 模型默认使用 CUDA，并同时配置离线识别模型与实时流式模型。
- 实时识别只走 X-ASR 原生流式会话，移除分块后反复调用离线 ASR 的伪流式与流式后二次离线识别。
- 离线识别增加兼容 CapsWriter-Offline 使用习惯的热词和正则替换规则。
- 移除多模型合并策略及其前后端数据结构和界面。
- 修复文件转写录音控制栏布局。
- Higgs TTS 同时支持本地服务和 Boson 远程 API，连接信息由用户配置。
- 增加开发调试台，集中展示 HTTP、WebSocket、ASR、TTS 等延时和错误。
- 对每项需求建立自动化或端到端验证，并输出测试报告。

## 实施步骤

1. 重构设置模型：使用独立的 `offlineEngine` 和 `streamingEngine`，迁移旧设置；所有 ASR 默认设备改为 CUDA；删除工作模式、多模型和合并策略字段。
2. 收敛后端转写接口为单离线模型；删除模型路由/合并逻辑和响应中的多引擎结果。
3. 按 X-ASR 示例重写实时会话：512 帧输入、约 0.7 秒预卷、VAD 上升沿建流、持续增量解码、下降沿补尾静音并结束；拒绝不支持原生流式的引擎。
4. 重写前端实时识别入口，统一连接 `/v1/stream`，删除录音分块后调用 `/v1/transcribe` 的伪流式实现。
5. 新增离线热词服务、动态文件加载、别名/黑名单/拼音近似替换和正则规则；在离线 ASR 结果返回前应用，并提供管理 API/UI。
6. 删除合并策略 UI，调整模型管理、工具栏和结果展示；修复文件转写底部录音按钮同排布局。
7. 扩展 Higgs TTS 本地/远程提供方配置，后端代理 Boson 鉴权和原始 PCM 流，增加不泄露 token 的连接测试。
8. 新增前端开发调试台与统一遥测，统计请求、流式连接、首个局部结果、最终结果、首包音频和总耗时。
9. 执行后端单元/集成测试、真实 X-ASR 流式测试、前端类型检查和构建；记录逐项证据到测试报告。
10. 更新 CHANGELOG、README 和 ASR/TTS/调试文档。

## 验收标准

- 模型管理同时显示离线模型和实时模型，初始配置全部为 CUDA，不再显示工作模式或合并策略。
- `/v1/transcribe` 仅接受单个离线模型；实时 WebSocket 不再执行离线分块识别或最终离线精修。
- X-ASR 在测试音频上产生多个 partial 和一个 final，尾部结果完整。
- 修改热词/规则后无需重启即可影响下一次离线识别，别名、黑名单和正则规则均有测试覆盖。
- 文件转写的录音、实时识别和网络状态在桌面宽度下保持同一行。
- Higgs 本地和 Boson 远程请求结构均通过代理测试，Authorization 不进入日志或响应。
- 调试台能实时显示主要链路延时并可清空、筛选和导出。
- 测试报告逐条映射九项需求到命令、结果和限制。
