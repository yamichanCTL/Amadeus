# 桌面语音识别确认、标点、遥测与 X-ASR 多窗口模型计划

> **父文档**: [← 返回文档索引](../README.md)
> **相关模块**: [桌面端](../desktop/README.md) · [ASR](../asrapp/asr/README.md)

## 任务目标

1. 将桌面端“文件转写”统一命名为“语音识别”。
2. 文件选择/拖放后仅进入待确认状态，用户点击确认按钮后才上传识别；删除“最近任务”卡片。
3. 修复标点恢复开关及后端处理链路，补充回归测试。
4. 完善开发调试台：按任务关联并可视化 ASR、TTS 和实时变声 VAD→ASR→TTS 的端到端阶段耗时，包含首个流式结果/首 token/首音频时间。
5. 删除暂不使用的说话人分离前后端入口、配置和实现。
6. 为 X-ASR 增加可下载、可选择的不同毫秒窗口模型，模型设置界面用单选方式切换；模型来源和文件格式以 Hugging Face 官方仓库实物为准。
7. 为每项需求执行针对性验证并输出测试报告。

## 实现步骤

1. 盘点现有转写、标点、遥测、实时 TTS、X-ASR 和说话人分离代码，确认工作区既有改动边界。
2. 修改语音识别页面的命名与待确认交互，移除最近任务区。
3. 修复标点恢复的配置传递、处理器加载/调用和结果返回，并增加测试。
4. 建立任务级 trace/span 遥测数据结构；给文件 ASR、流式 ASR、TTS、实时 VAD→ASR→TTS 补齐关键阶段埋点，在调试台展示瀑布图、首结果指标及汇总。
5. 删除说话人分离实现、请求字段、UI 开关和无效依赖，保持旧持久化数据兼容清理。
6. 核对 Hugging Face X-ASR 模型变体，扩展后端模型目录/下载信息/切换参数以及前端单选设置。
7. 逐项执行静态检查、单元/业务测试、生产构建和可运行的端到端验证；记录环境限制。
8. 更新模块文档、`doc/CHANGELOG.md` 和专项测试报告。

## 风险评估

- 工作区已有未提交改动：只做局部补丁并在提交前检查 diff，不回退无关内容。
- 不同 X-ASR 窗口模型可能存在不同文件名或 runtime 参数：必须按 Hugging Face 仓库元数据校验，不凭名称推断。
- CUDA/真实麦克风/播放设备可能无法在当前环境自动化验证：协议、事件顺序和耗时计算可自动验证，硬件声音链路需单独标注。
- 标点模型可能未下载或第三方依赖不可用：处理器应明确报告降级/错误，测试覆盖正常路径和无模型路径。
- 遥测不能阻塞音频实时链路：前端仅使用高精度时间戳和内存事件，不在音频回调中执行重 IO。

## 执行记录

- 标点实现确定为项目已有依赖 FunASR 的 `ct-punc`，替换原占位透传；单元测试使用可控模型边界，真实首次权重下载因当前外部执行额度被拒绝，测试报告单独标注。
- Hugging Face 当前官方仓库明确提供 160 / 480 / 960 / 1920 ms 四套模型，已全部下载并用同一段音频逐套完成 CPU online decode。
- CUDA runtime 本身仍为 `sherpa-onnx 1.13.2+cuda12.cudnn9`，但本轮受限进程无法访问 NVML/GPU，四窗口改用 CPU 实机解码验证；未复写此前 CUDA 基线结论。
- 调试台沿用已有 500 条内存事件，在其上增加 trace/stage 数据和最近 12 个任务瀑布图，避免引入新的持久化或后端遥测服务。
