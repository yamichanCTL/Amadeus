# 桌面 ASR 总结、隐私、模型发现与退出输入可靠性改进 Plan

> **父文档**: [返回计划索引](README.md)
> **相关文档**: [桌面端总览](../desktop/README.md) · [后端总览](../asrapp/backend/README.md)

## 任务目标

完成并验证以下八项改进：

1. 后端日志记录 ASR 经 AI 润色后的最终文本。
2. 当日总结的记录类型改为“离线识别 / 实时识别”选择，不再要求手输类别。
3. 当日总结开始时间默认当天 `00:00`，结束时间默认当前本地时间。
4. 当日总结增加可持久化 Prompt 设置，并随主动、被动总结请求发送。
5. 未允许服务端保存调试数据时，服务端不得归档调试音频/JSON；用户本机 Electron 归档不属于服务端留存并保持现有行为。
6. 桌面模型管理完全使用后端返回的 ASR 模型能力列表，新增后端引擎无需修改前端枚举。
7. 未选择后台运行时，窗口“关闭”必须真正退出 Electron 进程。
8. 加固 Windows 重启后的文本自动填充：输入助手失效时自动重建并重试，同时保留剪贴板与可复制结果兜底。

## 影响范围分析

- 后端 ASR：`backend/app/tasks/asr_task.py`、模型注册/列表 API、相关测试。
- 后端总结与归档：`backend/app/core/archive.py`、`backend/app/core/llm.py`、schema/API 测试。
- 桌面总结与设置：`frontend/desktop/src/pages/Summary.tsx`、`App.tsx`、`useASRStore.ts`、API 类型与测试。
- 桌面隐私：`recordingService.ts` 继续区分本机归档与服务端调试留存，服务端开关只随请求传递。
- Electron 生命周期与输入：`electron/main.ts` 及可独立测试的决策/重试辅助模块。
- 文档：桌面使用说明、后端 API/架构说明、测试报告和 `doc/CHANGELOG.md`。

## 实现步骤

1. 为时间默认值、总结类别映射、隐私归档门控、关闭决策和输入助手重试添加回归测试。
2. 后端在 LLM 润色成功后输出结构化日志（任务 ID、字符数、润色文本），避免记录 API Token。
3. 总结页使用固定类别选项并设置本地时间默认值；新增 `summaryPrompt` 设置，主动/被动请求统一使用。
4. 将“允许服务端保存调试数据”作为后端成功/失败归档入口的硬门槛；关闭时服务端不创建音频/JSON 文件。
5. 使用后端现有 `/v1/models` 注册表与 `model_modes` 元数据，前端动态展示和初始化配置，不再维护引擎枚举。
6. 将后台运行偏好由 renderer 明确同步给 main process；关闭事件直接依据同步状态决定隐藏或退出。
7. 文本输入助手在进程失效、未就绪或首次注入失败时清理并重新拉起，最多重试一次。
8. 运行目标 Vitest/Pytest、TypeScript、Vite、Python compileall、文档构建和 `git diff --check`。

## 风险评估

- **隐私兼容**：关闭归档后历史仍保留当前会话展示，但不会产生本地调试音频/JSON；既有文件不自动删除。
- **模型发现**：导入第三方引擎模块可能失败，发现逻辑必须隔离单个模块异常并记录警告，不能阻断后端启动。
- **退出行为**：退出会终止热键、托盘和录音；仅当用户明确启用后台运行才隐藏窗口。
- **Windows 输入**：Linux 环境只能验证状态机、构建和模拟失败重试；真实 Windows 重启后的目标窗口粘贴仍需 Windows E2E 复验。
- **日志隐私**：用户明确要求记录润色结果，因此只记录结果文本和任务元数据，绝不记录 Token 或完整请求配置。
